# Brand3 Perceptual Pattern Registry

Generated: 2026-05-15
Status: draft registry
Source: first three FLOC* perceptual case records and cross-case audit
Scope: registry only

## Purpose

This registry turns recurring observations from Charms, D4DATA, and Grandvalira Resorts into reusable Brand3/FLOC* perceptual patterns. It does not modify schema, scoring, prompts, reports, runtime, Visual Signature, or case records.

The registry is intentionally conservative. A pattern only becomes reusable when it can be expressed without importing a case-specific slogan or metaphor as a general rule.

## Registered Patterns

### `pattern_category_surface_translation`

**Pattern name:** Category-To-Surface Translation
**Confidence:** high
**Supporting cases:** Charms, D4DATA, Grandvalira

**Description:** An abstract domain is made perceptually legible through a surface system. The pattern appears when a case translates category complexity into a visible or stated behavioral frame.

**Perceptual meaning:** The brand does not merely state a category. It gives the category a readable surface logic.

**Observable signals:**

- The source names an abstract domain such as onchain games, decentralized data economy, or multi-resort ecosystem.
- The record identifies a perceptual organizing frame: ritual, motion metaphor, design-system governance.
- The case links strategy or identity to surface behavior.

**Typical tensions:**

- stated category / surface expression
- concept / implementation proof
- copy claim / visible evidence

**Emotional temperature:** controlled, intentional, concept-led.

**Common contexts:** Web3, technical platforms, multi-brand systems, identity systems, design sprints.

**Anti-patterns:**

- Treating the category itself as the perceptual pattern.
- Repeating the source metaphor without observing surface behavior.
- Turning concept translation into a quality claim.

**Perceptual flags:** premium: true; editorial: varies; ritualized: varies; cinematic: false; operational: true; playful: varies; restrained: varies; dense: false; atmospheric: varies; system-driven: true.

### `pattern_evidence_bound_behavior`

**Pattern name:** Evidence-Bound Surface Behavior
**Confidence:** high
**Supporting cases:** Charms, D4DATA, Grandvalira

**Description:** A perceptual reading is constrained by its evidence status: direct visual observation, copy-based claim, corroborating anchor, or weak inference.

**Perceptual meaning:** The pattern protects Brand3 from converting source language into confirmed visual truth.

**Observable signals:**

- Records separate extracted facts, visual observations, interpretations, and low-confidence inferences.
- Medium-confidence observations are often copy-based.
- Low-confidence inferences require human review.
- Contradictions name the boundary between claim and observed or unverified signal.

**Typical tensions:**

- copy-stated behavior / observed behavior
- strategic interpretation / evidence anchor
- review-ready pattern / downstream use

**Emotional temperature:** restrained, diagnostic, cautious.

**Common contexts:** all perceptual records, audits, review workflows, prompt design.

**Anti-patterns:**

- Calling a claim visible before a capture exists.
- Treating public case language as product behavior.
- Allowing low-confidence strategic intent into reusable language without review.

**Perceptual flags:** premium: false; editorial: true; ritualized: false; cinematic: false; operational: true; playful: false; restrained: true; dense: false; atmospheric: false; system-driven: true.

### `pattern_guided_movement`

**Pattern name:** Guided Movement
**Confidence:** medium
**Supporting cases:** Charms, D4DATA, Grandvalira

**Description:** The case frames movement through a guided path: gallery pacing, seamless UI guidance, or a design system that guides multiple identities.

**Perceptual meaning:** Movement is not only animation. It can be how a surface directs attention, sequence, identity, or system use.

**Observable signals:**

- Path, guiding, threshold, transition, seamlessness, or navigation language.
- Sequential gallery or lookbook-like movement.
- UI claims about guiding users.
- System claims about guiding multiple brand instances.

**Typical tensions:**

- guidance / intensity
- guided path / evidence gap
- smooth movement / complex system

**Emotional temperature:** controlled, directional, calm-to-dynamic depending on case.

**Common contexts:** case-study pages, onboarding, motion-led interfaces, design systems, multi-step product flows.

**Anti-patterns:**

- Treating any animation as guided movement.
- Treating the word "guide" as proof of interface clarity.
- Ignoring friction if the path is claimed but not visible.

**Perceptual flags:** premium: varies; editorial: true; ritualized: varies; cinematic: varies; operational: true; playful: varies; restrained: varies; dense: false; atmospheric: varies; system-driven: true.

### `pattern_concept_bearing_motion`

**Pattern name:** Concept-Bearing Motion
**Confidence:** medium
**Supporting cases:** D4DATA, Charms

**Description:** Motion or implied motion is used to carry a concept rather than to decorate the surface.

**Perceptual meaning:** Movement becomes part of identity logic when it embodies the case's conceptual source.

**Observable signals:**

- Motion production language tied to a concept.
- Transitions described as meaningful, not merely polished.
- Surface rhythm or repeated pacing that supports a conceptual atmosphere.
- Human review required when motion is only stated in copy.

**Typical tensions:**

- motion claim / motion evidence
- cinematic intensity / interface guidance
- production method / perceptual effect

**Emotional temperature:** dynamic, cinematic, potentially intense.

**Common contexts:** motion-led identity, campaign surfaces, case-study galleries, Web3 products, audiovisual systems.

**Anti-patterns:**

- Treating production tools as perceptual evidence by themselves.
- Calling motion concept-bearing without showing what concept it carries.
- Overusing cinematic language for static surfaces.

**Perceptual flags:** premium: varies; editorial: false; ritualized: varies; cinematic: true; operational: false; playful: false; restrained: false; dense: varies; atmospheric: varies; system-driven: varies.

### `pattern_system_cohesion_difference`

**Pattern name:** Cohesion With Preserved Difference
**Confidence:** medium
**Supporting cases:** Grandvalira, Charms

**Description:** A system creates recognizable unity while preserving meaningful variation. In system-heavy cases, this appears as multi-brand cohesion; in softer identity cases, it appears as repeated motifs without total rigidity.

**Perceptual meaning:** Strong systems do not always remove difference. They can make difference legible inside a controlled frame.

**Observable signals:**

- Multi-brand components, foundations, shared structures, or repeated motifs.
- Explicit language about cohesion and unique personalities.
- Repetition with controlled variation.
- Need for variant, component, or sub-brand evidence.

**Typical tensions:**

- unity / differentiation
- cohesion / uniqueness
- modularity claim / component evidence

**Emotional temperature:** ordered, institutional, controlled.

**Common contexts:** design systems, multi-brand portfolios, product ecosystems, scalable identity systems.

**Anti-patterns:**

- Treating uniformity as coherence.
- Treating uniqueness claims as proven without variant evidence.
- Calling repetition modular without component proof.

**Perceptual flags:** premium: true; editorial: false; ritualized: false; cinematic: false; operational: true; playful: false; restrained: true; dense: varies; atmospheric: false; system-driven: true.

### `pattern_claim_signal_gap`

**Pattern name:** Claim / Signal Gap
**Confidence:** high
**Supporting cases:** Charms, D4DATA, Grandvalira

**Description:** A productive or unresolved gap appears between what the source claims and what the available evidence can support.

**Perceptual meaning:** The gap is not automatically a flaw. It is a review object that prevents premature generalization.

**Observable signals:**

- Contradictions marked as soft or unverified.
- Claims such as modularity, seamless guidance, scalability, or cohesion that need more evidence.
- Low-confidence inferences that require human review.
- Friction log entries about copy-only or system evidence.

**Typical tensions:**

- claim / evidence gap
- stated intent / observed signal
- source language / Brand3 interpretation

**Emotional temperature:** diagnostic, restrained, skeptical.

**Common contexts:** public case pages, source-led audits, early corpus construction, prompt QA.

**Anti-patterns:**

- Treating every gap as a negative contradiction.
- Letting uncertainty disappear in polished prose.
- Resolving gaps by adding unsupported strategy language.

**Perceptual flags:** premium: false; editorial: true; ritualized: false; cinematic: false; operational: true; playful: false; restrained: true; dense: false; atmospheric: false; system-driven: true.

### `pattern_threshold_pacing`

**Pattern name:** Threshold Pacing
**Confidence:** medium
**Supporting cases:** Charms, D4DATA

**Description:** The surface creates a felt transition from one state into another: gallery to action, concept to interface, calm surface to intense motion, or case world to external product.

**Perceptual meaning:** Pacing can behave like a threshold when it changes how the viewer enters, exits, or interprets a system.

**Observable signals:**

- Gallery-first sequencing.
- Delayed or threshold-like CTA.
- Transition language.
- Motion or rhythm that marks passage between states.

**Typical tensions:**

- contemplation / action
- intensity / control
- atmosphere / utility

**Emotional temperature:** anticipatory, atmospheric, controlled.

**Common contexts:** case-study pages, launch pages, motion-led identity surfaces, campaign-to-product transitions.

**Anti-patterns:**

- Treating any scroll sequence as threshold behavior.
- Calling slow pacing premium without evidence.
- Confusing atmospheric delay with missing product clarity.

**Perceptual flags:** premium: true; editorial: true; ritualized: true; cinematic: true; operational: false; playful: varies; restrained: true; dense: false; atmospheric: true; system-driven: false.

## Unstable Patterns

These patterns are promising but not stable enough for broad reuse:

- `concept_bearing_motion`: needs actual motion capture, not only production-method copy.
- `guided_movement`: currently mixes interface guidance, case-page pacing, and system guidance.
- `system_governance`: strong for Grandvalira, but needs more system cases and component evidence.
- `premium_restraint`: well supported in Charms, weaker in D4DATA.
- `threshold_pacing`: strong in Charms, still speculative outside case-page flows.

## Patterns That May Be Too Case-Specific

- Kendo-as-data-economy discipline.
- Datamosh as disruption.
- MetaHuman as humanizing layer.
- Infinite winter as system atmosphere.
- Agentic games as soft social ritual.
- The three-resort unity/difference structure.

These should remain examples unless future cases repeat the underlying behavior without relying on the same metaphor.

## Vocabulary Collisions

- `system`: can mean visual identity system, product system, design system, ecosystem, or governance layer.
- `motion`: can mean animation, transition, production method, cadence, or implied movement.
- `guidance`: can mean UI guidance, case-page pacing, strategic direction, or system governance.
- `modular`: can mean component architecture, narrative flexibility, or brand extensibility.
- `cohesion`: can mean visual consistency, multi-brand governance, or narrative unity.
- `ritual`: can mean repeated interaction, emotional cadence, or merely poetic copy.
- `premium`: can mean restraint, polish, aspiration, or simply high production value.

## Language That Still Sounds Too Generic Or LLM-Like

Avoid promoting these phrases into registry language unless they are made specific with evidence:

- `forward-thinking`
- `seamless experience`
- `user-friendly`
- `impactful`
- `powerful`
- `innovative`
- `ecosystem`
- `digital universe`
- `refined`
- `intuitive`
- `meaningful connection`
- `positive impact`

These words can remain source facts, but Brand3 should translate them into observed behaviors before using them in reports or reviews.

## Registry Boundary

This registry is not a scoring rubric. It is not a prompt file. It is not Visual Signature guidance. It is a controlled vocabulary and pattern layer for future human review.
